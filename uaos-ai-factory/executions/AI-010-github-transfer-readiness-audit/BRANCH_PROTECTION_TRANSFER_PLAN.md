# Branch Protection Transfer Plan

Status: local plan only

Before transfer completion, prepare a target branch policy that keeps `master` protected, requires review for risky changes, and blocks direct release work until staging gates exist.

After transfer, the owner should manually confirm:

- Default branch name.
- Required checks.
- Review requirements.
- Admin bypass policy.
- Issue and PR templates.
- Safety workflow status.

No repository setting was changed by this plan.
