# Screenshot / Send Pack Cleanup Index

LOCAL ONLY - OWNER REVIEW ONLY - NO SENDING

## Purpose

This index consolidates the safe screenshot and send-pack guidance for Jobcenter and supporter review. It does not send messages, create public URLs, create production files, run deploy tools, or claim real keyboard export support.

## Use These Labels

- `LOCAL ONLY`
- `STATIC LOCAL DEMO`
- `OWNER REVIEW ONLY`
- `NOT PUBLIC RELEASE`
- `NOT PRODUCTION`
- `NO PAYMENT / LIVE CUSTOMER ACTIVITY`
- `NO REAL KEYBOARD EXPORT`

## Screenshots To Capture Manually

Capture only local/private screenshots that show:

- Owner dashboard or local project overview.
- Private/static demo gateway page.
- Jobcenter-safe presentation context.
- Supporter-safe presentation context.
- Screenshot / visual proof pack index.
- Final send checklist.
- Message draft review seal.
- Owner-neutral package review summary, if relevant to the recipient.

Do not automate capture, upload screenshots, or create public links in this task.

## What Not To Claim

Do not claim:

- Public release.
- Production readiness.
- Live customer use.
- Payment readiness or active payment flow.
- Public URL availability.
- Real keyboard writer/export compatibility.
- `.SET` or `.STY` export support.
- Ownership of Kontakt, Native Instruments, sample libraries, or proprietary content.
- External automation readiness.

## Jobcenter-Safe Wording

Use cautious wording:

`This is a local prototype and project planning package for UAOS / AE Platform. It shows current progress, local demo screens, and the next safe development steps. It is not public, not production, and not payment-ready. Real keyboard export and keyboard transfer remain blocked.`

## Supporter-Safe Wording

Use private support wording:

`This is a private local demo and owner review package. It can be used to understand the project direction and current prototype status. It is not a public product, not a live service, and not ready for customer or keyboard use.`

## Attachment Checklist

Before attaching anything, the owner manually verifies:

- Each file is needed for the recipient.
- Each screenshot has a local-only or static-local-demo label.
- Each message says the project is private/local/prototype status.
- No attachment claims public launch, production, payment, live customers, real keyboard export, or keyboard transfer.
- No proprietary samples, Kontakt, Native Instruments, or sample library content are attached.
- No `.STY`, `.SET`, `.PRS`, `.STL`, `.PAT`, `.MSP`, or `.KST` file is attached.
