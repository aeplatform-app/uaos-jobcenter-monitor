# Owner Send-Ready Review Steps

LOCAL ONLY - MANUAL REVIEW ONLY - NO SENDING

## Step 1: Choose Recipient Context

- Jobcenter: funding / planning / project progress context.
- Supporter: private explanation / support context.

Do not use the same wording for every recipient. Keep the message truthful and limited.

## Step 2: Select Minimal Attachments

Attach only the smallest useful set:

- One short project overview or business-plan note.
- One or more local/static screenshots.
- Optional safe message draft.
- Optional owner-neutral review summary if it helps explain the musical planning layer.

## Step 3: Review Screenshot Claims

Each screenshot must clearly show or be captioned as:

- `LOCAL ONLY`, or
- `STATIC LOCAL DEMO`, and
- `NOT PUBLIC RELEASE`.

Reject any screenshot that implies public launch, production use, payment/live customer activity, real keyboard export, keyboard transfer, or proprietary sample ownership.

## Step 4: Review Message Text

Jobcenter-safe wording:

`The attached material shows a local prototype and planning status for UAOS / AE Platform. It is intended for private review of the project direction, current progress, and realistic next steps. It is not a public release, production service, payment system, or real keyboard export.`

Supporter-safe wording:

`The attached material is a private local review pack. It shows the current prototype direction and what still needs to be built safely. It is not a public product and should not be shared as a live service.`

## Step 5: Final Attachment Checklist

- Local-only labels are visible.
- No public URL claim is present.
- No payment/live customer claim is present.
- No production release claim is present.
- No real keyboard export claim is present.
- No keyboard transfer is implied.
- No proprietary samples or Kontakt / Native Instruments content is included.
- No keyboard-native file extensions are attached.

## Step 6: Manual Owner Decision

Only the owner decides whether to send materials outside the repo. Codex must not send, schedule, upload, deploy, or automate external delivery from this task.
