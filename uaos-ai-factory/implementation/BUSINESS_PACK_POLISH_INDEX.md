## Purpose

This index polishes the local Jobcenter/supporter business-pack references into one owner-facing review path. It does not send anything, create public URLs, create production release files, or approve payment/live customer claims.

## What To Attach

Only after owner review, attach the minimum useful materials:

- Short Jobcenter or supporter message text.
- Local prototype/project overview.
- Local/static screenshots with `LOCAL ONLY - NOT PUBLIC RELEASE` captions.
- Funding or workstation/laptop need note, if relevant.
- Owner dashboard or final handoff start page.
- Neutral package summary only if it helps explain the music-planning layer.

## What Not To Attach

Do not attach:

- App source code or frontend source code.
- Deploy instructions or Vercel output.
- Payment setup material.
- Public release claims.
- Real keyboard export files.
- `.STY`, `.SET`, `.PRS`, `.STL`, `.PAT`, `.MSP`, or `.KST`.
- Kontakt, Native Instruments, sample library, audio asset, or proprietary content.
- Legacy `.STY` inventory files as deliverables.

## Local Prototype Wording

Use:

`This is a local prototype and planning package. It shows current project direction, local review materials, safety boundaries, and next development steps. It is not public, not production, and not payment-ready.`

## Laptop / Workstation Need Wording

Use:

`A suitable laptop/workstation is needed to continue local development, testing, documentation, and safe demo preparation. This request supports prototype development and review work; it is not a claim that the platform is already production-ready.`

## Owner Review Before Sending

Before any manual send, the owner checks:

- Recipient context is correct.
- Attachments are minimal.
- Captions use local-only wording.
- No public release, payment, production, real keyboard export, or proprietary library/sample claim is present.
- No external automation is used.
