## What Not To Claim

Do not claim:

- Public launch.
- Production readiness.
- Live customer use.
- Payment readiness.
- Public URL availability.
- GitHub transfer completion.
- Vercel deployment.
- External automation approval.
- Real keyboard writer/export readiness.
- Proprietary sample/library ownership.

## What Not To Send

Do not send:

- App.jsx.
- Frontend source code.
- App source files.
- Deploy commands or deploy instructions.
- Vercel command output.
- Payment setup material.
- Production release files.
- Real keyboard writer/export files.
- Real `.STY`, `.SET`, `.PRS`, `.STL`, `.PAT`, `.MSP`, or `.KST` files.
- Kontakt, Native Instruments, sample library, audio asset, or proprietary content.

## Final Owner Review Steps Before Sending

1. Pick recipient type: Jobcenter or supporter/friend.
2. Select only the minimum needed attachments.
3. Check every attachment title and caption for local-only wording.
4. Remove anything that implies public release, payment, production, or real export.
5. Confirm no source code or proprietary content is attached.
6. Confirm current remote remains unchanged.
7. Manually approve the final text before sending.

## Next Safe Action After Sending

After sending manually, create a local-only follow-up note recording:

- Recipient type.
- Date sent.
- Files actually sent.
- Questions received.
- Next owner decision.

Do not automate follow-up. Do not enable external automation.
