# Visual QA Selected Package Panel Acceptance Criteria

LOCAL ONLY - ACCEPTANCE CRITERIA - NO UI CHANGE

## Pass Criteria

The selected package panel passes manual visual QA only if all of the following are true:

- Panel is visible.
- `Selected Neutral Package` title is visible.
- `owner-neutral-003` is visible.
- `LOCAL ONLY` is visible.
- `READ ONLY` is visible.
- `NOT KEYBOARD OUTPUT` is visible.
- `Validation: PASS` is visible.
- `Real keyboard output: NO` is visible.
- `Keyboard transfer: NO` is visible.
- No export button appears in the panel.
- No payment action appears in the panel.
- No deploy/public wording appears in the panel.
- No keyboard compatibility claim appears in the panel.
- Responsive layout is readable.
- Dark mode/readability is acceptable.

## Fail Criteria

Fail manual visual QA if any of these appear:

- Ready for keyboard wording.
- Export to organ wording.
- Production or live release wording.
- Payment enabled wording.
- Real writer ready wording.
- Keyboard-native compatibility claim.
- Keyboard transfer action.
- Export button.
- Public deploy action.

## Safety Result

Passing this checklist does not approve keyboard output, keyboard transfer, deployment, payment flows, public URLs, or external automation.
