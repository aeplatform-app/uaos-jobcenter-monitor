# Owner Neutral 003 Build Requirements

LOCAL ONLY - REQUIREMENTS ONLY - NO BUILD OUTPUT

## Build Status

`owner-neutral-003` is not generated by SAFE-012. This file defines requirements for a later owner-approved neutral metadata build only.

## Required Inputs

- `owner-neutral-002` JSON as the source package.
- Owner feedback form or owner written approval.
- SAFE-012 planning report.
- Current safety checks passing.

## Required Package Constraints

- File extension must be `.uaos-neutral.json`.
- Package must be JSON metadata only.
- `keyboard_native` must be `false`.
- `real_keyboard_compatibility` must be `UNVERIFIED`.
- `owner_only` must be `true`.
- `not_public_release` must be `true`.
- `not_production` must be `true`.
- `no_push`, `no_deploy`, `no_vercel`, and `no_payment` must be `true`.
- No audio, samples, MIDI payload, keyboard-native pattern data, or proprietary content.

## Required Section Plan

Preferred sections:

1. `Intro - gentle setup`
2. `Main Groove - verse foundation`
3. `Lift - chorus energy`
4. `Transition - short break`
5. `Outro - safe close`

Each section must include:

- Stable section ID.
- Order number.
- Metadata-only flag.
- Owner-review note.

## Required Track Plan

Preferred tracks:

1. `Lead Line Planning Metadata`
2. `Harmony Bed Planning Metadata`
3. `Rhythm Feel Planning Metadata`

Each track must include:

- Stable track ID.
- Role.
- Instrument family hint.
- Metadata-only flag.
- No sample/library reference.

## Required Metadata Plan

Include:

- Source package ID: `owner-neutral-002`.
- Revision reason.
- Owner feedback source.
- Owner feedback status.
- Review round.
- Tempo and tempo range.
- Key center.
- Time signature.
- Mood.
- Style intent.
- Safety notes.
- Rollback notes.

## Required Chord / Style Placeholder Plan

Placeholders may include:

- Human-readable chord text.
- Section references.
- Groove intent text.
- Style family hint.

Placeholders must also include:

- `metadata_only: true`
- `not_keyboard_pattern_data: true`
- `not_style_file_content: true`
- No copied style data.
- No legacy `.STY` material.

## Required Validation Plan

Before sealing a future build, validate:

- Package exists.
- Package ID exists.
- Source package ID exists.
- Required labels exist.
- Required safety booleans are correct.
- Sections exist.
- Tracks exist.
- Metadata exists.
- Chord/style placeholders are metadata-only.
- Manifest/checksum references exist.
- No keyboard-native extensions are produced.
- No proprietary samples or Kontakt / Native Instruments content exists.
- Remote remains `https://github.com/Sari-raslan/universal-arranger-os.git`.

- `npm run ai:factory:qa-command-dashboard`

## Build Stop Conditions

Stop immediately if any request asks for:

- `.SET`, `.STY`, `.PRS`, `.STL`, `.PAT`, `.MSP`, or `.KST`.
- Real keyboard output.
- Keyboard transfer.
- Legacy file movement, deletion, restore, copying, or content reading.
- Proprietary samples.
- Kontakt / Native Instruments content.
- Push, deploy, Vercel, payment, public release, or external automation.
