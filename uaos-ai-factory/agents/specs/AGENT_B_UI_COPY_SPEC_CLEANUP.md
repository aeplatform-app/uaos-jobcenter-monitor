Status: LOCAL ONLY

Purpose: provide safer product wording and page text specifications for AE Platform / UAOS without touching app source files.

No source-code implementation is included or allowed by this spec.

## Clean AE Platform / UAOS Wording

Use:

- AE Platform is the local owner/project umbrella.
- UAOS means Universal Arranger OS.
- UAOS is a prototype/planning system for arranger keyboard workflows, demo materials, and future product planning.
- Current materials are local-only and are not a public release.
- Current demos are presentation aids, not live public services.

Avoid:

- Claims that the product is launched.
- Claims that production service is ready.
- Claims that payment or customer onboarding is live.
- Claims that real keyboard format export is ready.
- Claims of ownership over third-party samples, Kontakt, Native Instruments, or commercial libraries.

## Jobcenter-Safe Demo Wording

Use:

- This is a local prototype and planning package.
- The goal is to show project direction, skills, documentation, and next milestones.
- Funding/support discussion should be milestone-based.
- Hardware/workstation/laptop needs are project enablement needs, not proof of public launch.
- Demo screenshots and local gateway pages are private presentation aids.

Avoid:

- "Launched product."
- "Production-ready platform."
- "Live customer system."
- "Payment-ready business."
- "Real keyboard export is complete."
- "Commercial library ownership."

## Supporter-Safe Demo Wording

Use:

- This is a private local demo for explanation and feedback.
- The project is moving through controlled local planning stages.
- Help is useful for equipment, documentation, presentation polish, and careful next steps.
- Any public or paid step needs explicit owner approval later.

Avoid:

- Asking supporters to treat the demo as a public product.
- Asking supporters to share public links.
- Saying the product is ready for customers.
- Saying payment, production release, or real export is active.

## Forbidden Claims List

Do not claim:

- Public launch is complete.
- Production release is complete.
- Live customers are active.
- Payment or paid subscriptions are active.
- Vercel or hosting publication has been executed.
- GitHub transfer has been completed.
- External automation is approved.
- Background agents are running.
- 24-hour agents are running.
- Open-ended agents are running.
- Real keyboard writer/export is active.
- Real `.STY`, `.SET`, `.PRS`, `.STL`, `.PAT`, `.MSP`, or `.KST` files are generated.
- Kontakt, Native Instruments, sample libraries, or proprietary sample content are included or owned.

## Replacement Wording For Risky Terms

| Risky term | Safer replacement |
| --- | --- |
| launch | local preview / private presentation / planned future release |
| production | local prototype / planning state / controlled demo |
| production-ready | presentation-ready for private review |
| payment | future paid-flow planning / funding discussion / not active |
| customer-ready | suitable for private feedback only |
| live export | export planning / export concept / not active |
| real export | format export planning / not enabled |
| public URL | local/static demo reference / private screenshot aid |
| automated agents | bounded local task packets / no external automation |
| finished product | prototype package / staged project plan |
