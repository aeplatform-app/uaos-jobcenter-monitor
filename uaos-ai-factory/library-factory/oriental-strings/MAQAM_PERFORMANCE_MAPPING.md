# Maqam Performance Mapping

## Verhalten

- Maqam-aware phrase behavior wird als Metadaten geplant.
- Hijaz, Nahawand, Bayati, Rast, Kurd und Saba werden als spätere Mapping-Kandidaten geführt.
- Quarter-tone planning bleibt Metadatenplanung und erzeugt keine Keyboard-native Datei.
- Ornamentik wird als Performance-Regel beschrieben, nicht als kopierte Library.

## Sichere Regeln

- Keine Kontakt-Inhalte.
- Keine kopierten Libraries.
- Keine kommerziellen Phrasen.
- Nur eigene spätere Aufnahmen, Synthese oder MIDI-Demos.
